AppBar Widget

An AppBar is a most commonly used widget in almost all the mobile apps. An AppBar is mostly as a child of Scaffold widget

An app bar consists of a toolbar and potentially other widgets, such as a TabBar and a FlexibleSpaceBar. App bars typically expose one or more common actions with IconButtons which are optionally followed by a PopupMenuButton for less common operations (sometimes called the "overflow menu").

App bars are typically used in the Scaffold.appBar property, which places the app bar as a fixed-height widget at the top of the screen. For a scrollable app bar, see SliverAppBar, which embeds an AppBar in a sliver for use in a CustomScrollView.

The AppBar displays the toolbar widgets, leading, title, and actions, above the bottom (if any). The bottom is usually used for a TabBar. If a flexibleSpace widget is specified then it is stacked behind the toolbar and the bottom widget. The following diagram shows where each of these slots appears in the toolbar when the writing language is left-to-right (e.g. English):

Notable parameters:

  • title: Displays the title of the page
  • actions: Displays the action buttons on the right side of the app bar

Usage: The following code snippet shows the usage of backgroundColor, appBar and the body properties of the Scaffold widget

Scaffold( backgroundColor: Colors.green, appBar: AppBar(title: Text('I am learning flutter')), actions: <Widget>[], body: Text('My flutter app') )


Complete main.dart demonstrating the usage of AppBar widget without any action buttons:

import 'package:flutter/material.dart'; void main() { runApp(MyApp()); } class MyApp extends StatelessWidget { @override Widget build(BuildContext context) { return MaterialApp( title: 'Flutter Demo', theme: ThemeData( primarySwatch: Colors.blue, ), home: MyHomePage(title: 'My First Flutter app'), ); } } class MyHomePage extends StatefulWidget { MyHomePage({Key key, this.title}) : super(key: key); final String title; @override _MyHomePageState createState() => _MyHomePageState(); } class _MyHomePageState extends State<MyHomePage> { @override Widget build(BuildContext context) { return Scaffold( backgroundColor: Colors.green, appBar: AppBar( title: Text('I am learning flutter'), actions: <Widget>[]), body: Text('My flutter app') ); } }

AppBar with action buttons

The actions parameter of the app bar can be used to create action buttons on the App Bar

Usage of actions parameter

actions: <Widget>[] // [] means list, so actions parameter takes a list of Widgets (like, Text, ElevatedButton etc)

The above code can also be written as:

actions: [],

Now, lets create App Bar with 3 action buttons (With 3 Text Widgets), so you can expand the above code as:

actions: [ Text('Button 1'), Text('Button 2'), Text('Button 3') ]

As you can see from the above code, the actions now have list of 3 Text Widgets
